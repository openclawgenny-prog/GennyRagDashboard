  - `SKILL.md`: Skill documentation and capabilities

  - `requirements.txt`: Python dependencies (requests)

- **scaffold**: Create new service projects under `workspace/services/<name>/`

- **GitHub Actions workflow**: Auto-builds Docker images on push, detects changed services

- Logs access to `~/.openclaw/credentials/access.log`
- Registry file: `~/.openclaw/credentials/registry.json`

1. GitHub Actions workflow now builds correct service with proper tags
2. Added `deploy` command for full end-to-end flow
3. `push_service()` returns GitHub Actions run_id
4. Fixed `portainer_pull_image` error handling (raises instead of returning bool)
5. Added port binding support to `portainer_start_container`

## HTTP 400 Error Diagnosis (March 4, 2026 - Late Afternoon)

### Issue
- User reported [400] errors in TUI with some models
- Logs analyzed: `/tmp/openclaw/openclaw-2026-03-04.log`

### Findings
**Two error patterns:**
1. `"HTTP 400: Client specified an invalid argument"` (from agent/embedded subsystem)
2. `"400 {\"message\":\"\",\"code\":\"invalid_request_body\"}"`

**Root causes:**
- GitHub Copilot models (especially `gpt-5-mini`) rejecting OpenClaw request format
- Likely issues:
  - Thinking/reasoning parameters not supported
  - Tool schema incompatibilities
  - Request body validation failures
  - Token limits (one compaction error: 92325 exceeds 64000)

### Recommended Fixes
1. Change default model: `openclaw config set defaultModel "github-copilot/claude-sonnet-4.5"`
2. Disable thinking for strict models: `/thinking off`
3. Update OpenClaw: `npm install -g openclaw@latest`

### Status
- User asked how to approve exec commands (approval flow explained)
- Awaiting user to apply one of the fixes and test
